# スライド デザインガイド（generate.py 設計時に必ず適用）

定評ある原則（Presentation Zen / Duarte / コンサル流メッセージ設計 / ノンデザイナーズの CRAP / 8pt グリッド）と、[anthropics/skills の pptx スキル](https://github.com/anthropics/skills/tree/main/skills/pptx)のデザイン原則・QA 哲学を、本キットのテンプレ制約（**色固定**・**フォント Arial(英数字)＋Noto Sans JP(和文)**・ゾーン `y=3.0〜12.8 / x=1.07〜24.33`・python-pptx・テンプレ固定装飾）向けに**実行可能なルール**へ翻訳したもの。SKILL の手順2で `slides.py` と一緒に読み、各本文ページの設計に適用する。3.5「設計品質バー」が"やってはいけない"、本書が"こう作る"。

> **フラット標準（本文コンポーネント）**：本文カードは色ベタの見出し帯・ドロップシャドウを使わず、**枠なし白カード＋アクセントチップ＋濃色テキスト見出し**で組む。`slides` の `card()` / `callout()` を基本単位とし、影は既定オフ（`shape_box(shadow=False)` が既定）。和文は Noto Sans JP（latin は Arial）。chrome（表紙・章扉・見出し・装飾）はテンプレ継承のまま触らない。

> 補足：anthropics pptx スキルの技術部分（PptxGenJS / unpack-pack / トピック別パレット選定 / フォントペアリング / タイトル 36–44pt 指定）は、本キットの「python-pptx＋雛形テンプレ・ブランド色固定・Arial 固定」ワークフローと競合するため**意図的に取り込んでいない**。本書に翻訳済みのデザイン原則・QA 哲学（§9・§8）のみを正とする。

## 1. 1スライド＝1メッセージ
- そのページで読み手に残す結論を1つに絞る。要素は全てその結論を支える。
- `configure_body` の title はテンプレ固定だが、**内容は「事実」より「結論文」**にする（例：×「監査ログ要件」 ○ 与えられた title を活かしつつ本文で結論を主張）。サブタイトル（説明行）に一文で要点を置く。**このタイトル下のリード文（コンテンツ先頭 `y=3.0` に置く1行サマリ）は本文色 `TEXT`（#434343＝Google スライド「暗いグレー 4」）で描く**（補助グレー `TEXT_MUTED` ではない。詳細は §6）。

## 2. 視覚階層は「サイズ → 太さ → 色」の順で付ける
- 強弱はまずサイズ差、次に bold、最後に色。色で階層を作らない。
- 本文スライドの推奨フォント ladder（pt・`T_*` 定数。デザインシステム px×0.75）：
  - ページタイトル `T_H1`=24（chrome 固定）／カード見出し `T_H2`=15 bold（**濃色テキスト** TEXT かアクセント色。色ベタ帯の反転白文字にしない）／本文 `T_BODY`=10.5（TEXT_MUTED）／補足・注釈 `T_CAPTION`=8.25（TEXT_MUTED）／番号チップ 18 bold WHITE／大数値タイル 17–22 bold PRIMARY
  - **最小 8pt。8pt 未満禁止**（表セルでも 8 まで）。
- 1ページの強調（色面・大数字・太字の主役）は**1〜2点まで**。全部強調は無強調と同じ。

## 3. グリッドと余白（余白は必ず意図を持たせる）
- 横の基準：左端 `X0=1.07`、有効幅 `23.26`。多カラムは等幅＋ガター `0.4〜0.6cm`。
- カード内パディング `0.3〜0.45cm`、ブロック間の縦ギャップ `0.2〜0.3cm` で一貫させる（バラバラの余白にしない）。
- **コンテンツゾーンを使い切る**。意図しない `>1.0cm` の死に余白（特に縦中央寄せの浮き）を残さない。要素が少なければ間隔・サイズを上げてゾーンを満たす。
- 整列は近接・整列の原則を守る：関連要素は近く、無関係は離す。左揃え基本、中央寄せは短いラベル/数値タイルのみ。

## 4. 整列はセル分割で作る（偽装整列の禁止）
- ラベル：値、表形式は `shape_box` のセルを分けて整列させる。全角スペース／コロンでの桁揃えは禁止（3.5 連動）。
- 表は `table()` 相当のグリッド（ヘッダ帯＋ゼブラ＋カテゴリ見出し行）で罫線 `BORDER`。
- 行・セル・カードに**重ねて置く小要素（バッジ・ラベル・アイコン等）は、親の高さから中央位置を計算して上下中央に置く**（例：`y + (row_h - badge_h) / 2`）。`y + 0.15` のような固定オフセットは親高を変えた途端ずれ、テキスト（`anchor=MIDDLE`）と縦位置が食い違って違和感の原因になる。固定値で「だいたい中央」にしない。

## 5. 箇条書きの規律 / 図解優先
- 1ブロックの箇条書きは**最大5〜6行**。超える・種類が混ざる場合は分割するか図解化する。
- 文末・粒度を揃える（パラレリズム）。3階層以上のネストにしない。
- 関係・構成・フローは文章の箇条書きにせず、**箱＋接続線＋方向（矢印）**で示す。手順はステップカード、比較は横並びカラム、指標は数値タイル（3.5 の型→構造表に従う）。
- 接続線・矢印は `connector(slide, x1, y1, x2, y2, *, color, end_arrow=True, ...)` で引く（直角の屈曲が要るなら `connector_type=MSO_CONNECTOR.ELBOW`）。装飾の線は禁止（§9）＝線は「つなぐ・示す」機能があるときだけ。
- **定量データ（推移・構成比・比較の数値）は表や箇条書きでなくグラフにする**：推移→`line_chart`、構成比→`pie_chart`、量の比較→`bar_chart`。配色は `CHART_PALETTE`（ブランド色）が既定。⚠ グラフ内部テキスト（軸・データラベル・凡例）は `validate` の文字照合対象外なので、文字潰れ・はみ出しは 5-b 視覚確認で必ず見る。
- 写真・ロゴ・スクリーンショットは `picture(slide, path, x, y, w=..., )`（片側省略でアスペクト比保持）。引き伸ばし・潰れに注意し、ゾーン内に収める。

## 6. 色運用の規律（ブランド色のみ・装飾目的の色は禁止）
- `PRIMARY`＝主役アクセント（チップ・重要バッジ・主要数値・キーテイクアウェイの左バー）。広い面の塗りつぶしや見出し帯に多用しない。
- `SECONDARY`＝構造・補助見出し。`SUCCESS`＝肯定/正常、`DANGER`＝否定/警告、`HIGHLIGHT`＝推奨バッジ（意味に対応させる）。
- 本文文字＝`TEXT`、サブ＝`TEXT_MUTED`、罫線＝`BORDER`、淡い強調面＝`PRIMARY_LIGHT`/`SURFACE`（広用しない）。
- **タイトル下のリード文（コンテンツ先頭の1行サマリ＝説明行）は必ず `TEXT`（#434343＝Google スライド「暗いグレー 4」）で描く**。ページの導入として読ませる主役テキストなので、カード本文・補足の補助グレー `TEXT_MUTED`（#6B6B6B）より一段濃いトーンにする。リード文＝`TEXT`／カード本文・補足・キャプション＝`TEXT_MUTED` を必ず区別する（リード文を `TEXT_MUTED` にしない）。
- 色は必ず**意味**を持たせる。同じ意味には同じ色、装飾だけの色付けはしない。

## 7. カード/アイコン・影（フラット標準）
- 本文カードは `card()` を基本単位にする：**枠なし（ヘアライン枠 `BORDER`）の白カード＋左上のアクセントチップ＋濃色テキスト見出し**。色ベタの見出し帯（反転白文字）は使わない（"今時でない"印象の主因）。アクセント色は小面積（チップ・kicker・数値）に限定。
- 角丸・間隔・パディングを全ページで揃える（反復の原則）。カード間ガター 0.6〜0.7cm、内パディング 0.5cm 目安。カード見出しは1行に収まる短いラベルにする（長いと2行化して本文と干渉）。
- 結論・キーテイクアウェイは反転した濃色帯（`SECONDARY` の色ベタ）ではなく `callout()`（**直角の淡色バー**＋左アクセントバー＋濃色テキスト）で示す。**下地は必ず直角の矩形**にする（角丸ピルの上に直角のアクセントバーを重ねると左端上下でコーナーがかみ合わず、バーと下地の間に三日月状の隙間が出て崩れる＝§3.5 の角丸×直角の入れ子禁止。`slides.callout()` は直角で実装済み）。
- アイコンは任意。入れるなら意味のある箇所に等サイズで、ブランド色 hex で着色（`add_icon`）。飾りで散らさない。
- **影は既定オフ（フラット）**。`shape_box` は `shadow=False` が既定でテーマのドロップシャドウを除去する。影は古い印象の最大要因なので原則付けない（どうしても立体を出したい時のみ `shadow=True`）。

## 8. セルフレビュー（5-b で必ず）
**前提の心構え（anthropics pptx QA 哲学）：「問題は必ずある。それを見つけるのが仕事」。** 最初の確認で指摘ゼロなら、それは"よく見ていない"だけ。完了宣言の前に最低1回は「指摘 → 修正 → 再確認」のサイクルを必ず通す（5-b で問題なしと即断しない）。

各本文ページについて自問する：
1. 結論は1つに絞れているか／見出し・サブタイトルで伝わるか
2. 階層がサイズ→太さ→色の順で付き、強調は1〜2点に収まっているか
3. ゾーンを使い切り、余白は意図的か（浮き・死に余白がないか）
4. 整列はセル分割か（偽装整列でないか）
5. 情報の型に合う構造か（箇条書き流し込みでないか／図解すべきを図解しているか）
6. 色は意味に対応し、ブランド色のみか
1つでも No なら 3.5 に従い再設計（完了不可）。

## 9. "汎用・AIっぽい"見た目の排除（anthropics pptx スキルより本キットの制約向けに翻訳）

本キットは色とフォントが固定なので「トピック別パレット選定」は適用しない。代わりに**固定パレットの"使い方"で凡庸さを排除する**。

- **色は均等配分しない（ドミナンス階層）**：1ページ内で1色（通常は `TEXT` の本文か白地）が視覚重量の **60〜70%** を占め、補助1〜2色＋鋭いアクセント1色（`PRIMARY`）に絞る。色を全要素に均等にばら撒くと一気に凡庸になる（§2・§6 と連動）。
- **全スライドに視覚要素を1つ以上**：図・チャート・アイコン・色面・構造図形のいずれかを必ず持たせる。テキストだけのスライドを作らない（3.5 アンチパターン連動）。
- **モチーフを1つ決めて全ページ反復**：例「番号/ラベルチップ＋枠なし白カード」「色丸の中のアイコン」「片側の太い境界バー」などから**1種を選び、資料全体で同じものを繰り返す**。ページごとに別の見せ方を発明しない（§7 反復の原則を強化）。
- **レイアウトを毎ページ変えない/同じにしない、のバランス**：「全ページ同一テンプレの箇条書き」も「ページごとにバラバラの構図」も凡庸。情報の型（3.5 の型→構造表）に従い、2カラム・アイコン+テキスト行・グリッド・数値タイル等を**型に応じて選び分ける**（型が同じなら見せ方も揃える＝反復、型が違えば構造を変える）。
- **本文は左揃え、中央寄せはタイトル/短いラベル/数値タイルのみ**（§3 と連動。本文の中央寄せは"AIっぽさ"の代表例）。
- **見出し下に自前の装飾アンダーラインを引かない**：これは "AI 生成の手癖" の典型。ただし**テンプレが描く装飾主色バーは意図された固定デザインなので残す**（消さない・足さない、テンプレ固定デザインに従う）。
- **低コントラストを作らない**：淡色面（`PRIMARY_LIGHT`/`SURFACE`）の上に淡いグレー文字、濃色面の上に濃いアイコンを直置きしない。アイコンは色丸の中に置く等でコントラストを確保。
