# 自社ブランド向けに作り変える（リブランド手順）

同梱の既定雛形は **K.S.ロジャースのブランド** に沿っているが、
**仕組み（`slides.py` のヘルパーと検査ゲート）はブランド非依存**で、
ブランド設定を差し替えれば自社向けに転用できる。エンジンのコードは変更不要。
リブランドは次の**2ステップ**で完結する。

1. **[`brand.py`](../brand.py) を編集**（色・フォント・既定雛形ファイル名の単一ソース）
2. **`templates/` に自社雛形 `.pptx` を置く**（現雛形と同じ構造を満たすもの）

> ⚠️ `templates/スライド雛形.pptx` には K.S.ロジャースのロゴ・ブランドデザインが
> 埋め込まれている。これは同社の資産であり MIT（コード）の対象外。**自社利用時は
> 必ず下記 2. で雛形ごと差し替える**こと（KSR ロゴを載せたまま配布しない）。

---

## 1. `brand.py` を編集する（色・フォント・雛形名の単一ソース）

色・フォント・既定雛形ファイル名は **`brand.py` の1ファイルに集約**されている。
ここを書き換えれば、グラフ・カード・チップ・コールアウト・見出しまで配色が一括で追従する
（`slides.py` が `brand.py` を読み込み再エクスポートするため、他のコードは変更不要）。

```python
# brand.py
from pptx.dml.color import RGBColor

# 色（役割ベースの定数名。値だけ差し替え、名前はスキル／サンプルが参照するので変えない）
PRIMARY       = RGBColor(0xEC, 0x67, 0x39)   # 主色 → 自社プライマリに
SECONDARY     = RGBColor(0x1F, 0x4E, 0x79)   # 補色 …
SUCCESS       = RGBColor(0x5E, 0x8B, 0x7E)   # 成功・正常
DANGER        = RGBColor(0xC2, 0x54, 0x50)   # 警告・否定
HIGHLIGHT     = RGBColor(0xB0, 0x85, 0x1B)   # 強調・推奨
# TEXT / TEXT_MUTED / BORDER / SURFACE / WHITE は中立色なので通常そのままでよい

# フォント
FONT    = "Arial"          # latin（英数字）
JP_FONT = "Noto Sans JP"   # 和文（ea/cs）→ 自社指定フォントに
JOSEFIN = "Josefin Sans"   # 章番号・表紙の装飾英字

# 既定雛形ファイル名（下記 2. で置いた .pptx のファイル名に合わせる）
TEMPLATE = "スライド雛形.pptx"
```

- **定数名（`PRIMARY` 等）は変えず、値だけ**差し替える（名前はスキル・サンプル・`slides.py` が参照）。
- グラフ配色 `CHART_PALETTE` は色定数から組まれるため、色を変えれば自動追従する。
- latin（`FONT`）を変える場合、検査ゲート `validate_fit` の横幅計測は **その実フォントを参照**する
  （`fc-match`＝fontconfig でファイル解決。要素ごとの `font=` 上書き＝章番号の `JOSEFIN` 等も反映）。
  **OS から見えるフォントにすること** — 未インストールだと fontconfig が代替フォントで計測し、その旨を
  stderr に WARN する（レンダラ側も同様に代替するため計測とは整合するが、意図した字形にはならない）。
  fontconfig が無い環境では標準フォントディレクトリを family 名で走査 → 最後に同梱 Arial にフォールバック。
  和文フォント未搭載 PC でのフォールバック注意は [`CLAUDE.md`](../CLAUDE.md) §既知の落とし穴 を参照。
- ただし `validate_fit` は近似（横幅ベース）で、**LibreOffice 固有の行分割**（例: 2桁数字チップの桁間改行）
  までは再現しない。フォント差し替え後の最終確認は `validate_render`（実レンダ）＋目視で行うこと。

## 2. 雛形 PowerPoint を差し替える（必須）

`templates/` に自社ロゴ・デザインの `.pptx` を置き、`brand.py` の `TEMPLATE` をその
ファイル名にする（同名で上書きするなら `TEMPLATE` の変更も不要）。

差し替える雛形は、**現雛形と同じレイアウト構成・プレースホルダを満たす**必要がある
（ヘルパーがレイアウト index とプレースホルダ位置に依存しているため）。最低限そろえるもの：

| 必要なもの | 使う場所 | 備考 |
|---|---|---|
| スライド 0（表紙・自社ロゴ＋テキストボックス） | `update_cover(prs.slides[0], ...)` | "御中" または "ご提案資料" を含むテキストボックスを書き換える。ロゴはこのスライドに置く |
| スライド 3（裏表紙・自社ロゴ） | `add_back_cover(prs)` | 雛形の idx 3 を複製して裏表紙にする |
| Layout 1（章扉） | `add_slide(L_CHAPTER)` + `configure_chapter()` | 全面塗りの章扉。番号・タイトル・サブタイトルのプレースホルダ |
| Layout 2（本文） | `add_slide(L_BODY)` + `configure_body()` | section label（y≈0.94）・title（y≈1.42）・ページ番号サークルのプレースホルダ |

レイアウト index（`L_BACK=0 / L_CHAPTER=1 / L_BODY=2 / L_COVER=3`）と Layout 2 の
プレースホルダ位置の厳密仕様は [`CLAUDE.md`](../CLAUDE.md) §雛形仕様 を参照。
レイアウト**名**は自由に付け替えてよい：検査ゲート（ゾーン検査）やページ番号の
「本文レイアウトか否か」の判定は名前ではなく index（`L_BODY`）から動的に解決するため、
改名しても自動で追従する。守るべきは index 構成とプレースホルダ位置。雛形の
スライドサイズ（既定 25.40 × 14.29 cm = 16:9）を変える場合は、`slides.py` の
`SLIDE_W_CM` / `SLIDE_H_CM` とコンテンツゾーン定数も合わせて見直す。

---

## やらないこと（仕組みは変えない）

- ヘルパー関数のシグネチャ・検査ゲートのロジックは変更不要。上記 1〜2 で完結する。
- スキル（`/slides-new` `/slides-gen`）も雛形・色・フォントに非依存なのでそのまま使える。
- レイアウトの index やプレースホルダ前提を崩すと `configure_*` / `update_cover` /
  `add_back_cover` が機能しなくなる。**雛形は「中身のデザイン」だけ替え、「構造」は保つ**。
